# i18n — Internationalization

**Responsibility:** JSON-driven translation engine providing multi-language UI string resolution with zero external dependencies.

## Architecture

[app/core/i18n.py](app/core/i18n.py)

No gettext, Babel, or Qt Linguist — one module-global dict + one lookup function. Adding a new language requires only a JSON file in `app/assets/locales/` and an entry in `AVAILABLE_LANGUAGES`.

## Locale files

- `app/assets/locales/en.json` — English translations
- `app/assets/locales/zh.json` — Chinese translations (default)
- Both files must share the same key set (enforced by module self-check)
- Keys use dot-separated namespaces: `common.*`, `main.*`, `settings.*`, `vm.*`, `foldergrid.*`, `objectlist.*`, `preview.*`, `thumb.*`, `launcher.*`, etc.

## API

| Function | Signature | Description |
|----------|-----------|-------------|
| `tr` | `(key: str, **fmt) -> str` | Look up key in active locale; fallback: en → raw key. Supports `str.format(**fmt)` interpolation. |
| `set_language` | `(lang: str) -> None` | Switch active language by loading locale JSON. Unknown codes fall back to `DEFAULT_LANGUAGE`. |
| `get_current_language` | `() -> str` | Return active language code |
| `_load_translations` | `(lang: str) -> dict` | Internal — load and validate locale JSON from disk |

## Constants

| Name | Value | Description |
|------|-------|-------------|
| `AVAILABLE_LANGUAGES` | `{"en": "English", "zh": "中文"}` | Code → human label |
| `DEFAULT_LANGUAGE` | `"zh"` | Chinese is the default |

## Fallback chain

```
Active locale JSON → en.json → raw key string
```

If the active locale file is missing or corrupt, the system falls back to `DEFAULT_LANGUAGE`. If a specific key is missing from the active locale, it falls back to the English translation. If neither exists, the raw key string is returned.

## Usage pattern

### Widget construction

All UI strings are resolved at widget construction time. Widgets call `tr()` in `__init__`:

```python
from app.core import i18n as _i18n
button.setText(_i18n.tr("common.cancel"))
```

### String interpolation

Keys may contain `{placeholder}` tokens filled via `str.format`:

```python
_i18n.tr("vm.moved", name="MyMod")  # → "Moved 'MyMod' successfully."
```

### Language switch

Language preference is stored in `config.json` under the `language` field (type `str`, default `"zh"`). `main.py` calls `set_language(config.language)` before constructing any widgets. Switching languages requires application restart because widgets resolve strings at init time.

### Import convention

All consumer modules import with the alias `_i18n`:

```python
from app.core import i18n as _i18n
```

## Self-check

The module contains a `if __name__ == "__main__"` block that asserts:
1. Key parity between `en.json` and `zh.json`
2. `tr()` returns correct translations for both languages
3. Format interpolation works with keyword arguments
4. Unknown language codes fall back to default
5. Missing translation keys return the raw key

Run with: `python -m app.core.i18n`

## Requirements

### Requirement: Key parity enforced across all locale files
All locale JSON files SHALL expose the exact same set of keys. The module self-check SHALL detect and report keys present in one file but missing in another.

#### Scenario: Adding a key to en.json without zh.json is caught
- **WHEN** a developer adds a key to `en.json` but forgets `zh.json`
- **THEN** the self-check (`python -m app.core.i18n`) SHALL fail with the set of missing keys

### Requirement: Language preference persists across sessions
The user's selected language SHALL be stored in `config.json` under the `language` field and restored on next launch before any widgets are constructed.

#### Scenario: Language is restored on startup
- **WHEN** the user selects English in settings and restarts the application
- **THEN** `main.py` reads `config.language == "en"` before constructing `MainWindow`
- **AND** all UI strings render in English

#### Scenario: Default language is Chinese
- **WHEN** the application is launched for the first time with no `config.json`
- **THEN** the default language is `"zh"` (Chinese)

### Requirement: Fallback chain never raises
The `tr()` function SHALL never raise an exception. Missing keys SHALL return the raw key string. Corrupt or missing locale files SHALL fall back to `DEFAULT_LANGUAGE`.

#### Scenario: Missing translation key returns raw key
- **WHEN** `tr("nonexistent.key")` is called
- **AND** the key does not exist in either the active locale or `en.json`
- **THEN** the function SHALL return `"nonexistent.key"` without raising
